Обновление вспомогательных данных может потребоваться при разработке.
Список причин размещен в форме.

1. Обычно обновление вспомогательных данных происходит при выполнении обработчиков обновления.
В первую очередь, это обязательные обработчики обновления (Версия = *).

Для оптимального (быстрого) обновления вспомогательных данных можно выполнить только обязательные
обработчики обновления (Версия = *). При этом учитывается изменение некоторых свойств конфигурации.
Если изменения этих свойств нет, то часть обновления выполнена не будет (оптимизация).

Чтобы выполнить только обязательные обработчики обновления (Версия = *):
а) укажите параметр запуска ЗапуститьОбновлениеИнформационнойБазы (в строке запуска /C или в конфигураторе);
б) включите константу "Запустить обновление информационной базы" и перезапустите программу
(не поддерживается в модели сервиса).

2. Может возникнуть ситуация (особенно при активной отладке), когда происходили ошибки выполнения.
В этом случае, часть данных может обновиться, а часть нет. Для гарантированного (полного) обновления всех
вспомогательных данных нужно выполнить обновление с помощью этой обработки (дольше оптимального обновления).
