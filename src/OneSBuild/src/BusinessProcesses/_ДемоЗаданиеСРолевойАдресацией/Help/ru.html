Предназначен для демонстрации примера использования ряда подсистем.

В подсистеме Бизнес-процессы и задачи бизнес-процесс позволяет ввести задачу и назначить ее на исполнителя с помощью персональной или ролевой адресации. Включает в себя точку действия Выполнить, которая записывает новую задачу исполнителю, и запуск вложенного бизнес-процесса для проверки выполнения.

В подсистеме Групповое изменение объектов бизнес-процесс демонстрирует возможность группового изменения реквизитов нескольких бизнес-процессов.

В подсистеме Префиксация объектов к номеру бизнес-процесса добавляется префикс информационной базы, а также демонстрируется переназначение номера, если бизнес-процесс был перемещен в другой период.

В подсистеме Работа с файлами элементы справочника используются в качестве объектов-владельцев файлов вложений. Для хранения файлов-вложений используется общий справочник Файлы.

В подсистеме Управление доступом бизнес-процесс показывает пример логики ограничения доступа с использованием наборов значений.

В подсистеме Управление доступом бизнес-процесс показывает пример различий в ограничениях для сеансов разного назначения (для пользователей и для внешних пользователей), пример ограничения с использованием наборов значений и пример разных ограничений прав Чтение и Изменение.

  • Для пользователей список ограничивается по-разному для прав Чтение и Изменение по разным видам доступа:
    • Чтение списка ограничивается по виду доступа Пользователи и видам доступа указанным для предмета (Демо: Заказ покупателя, Демо: Счет на оплату покупателю, Пользователи, Файлы):
      • Пользователи (проверяется значение реквизита Автор).
      • Пользователи (проверяется значение реквизита Исполнитель, либо значения указанные для роли исполнителя).
      • Пользователи (проверяется значение реквизита Проверяющий, либо значения указанные для роли проверяющего).
      • <Виды доступа предмета> - см. соответствующие предметы.
      • Требуется, чтобы в одной из групп доступа было разрешено Автор или Исполнитель или Проверяющий или:
        • (когда предмет - это пользователь) предмет-пользователь или одно из значений, указанных для роли исполнителя или одно из значений, указанных для роли проверяющего;
        • (когда предмет пустой) одно из значений, указанных для роли исполнителя или одно из значений, указанных для роли проверяющего;
        •  (когда предмет не пользователь) предмет и одно из значений, указанных для роли исполнителя и одно из значений, указанных для роли проверяющего.
    • Изменение списка ограничивается по одному виду доступа:
      • Пользователи (проверяется значение реквизита Автор).
  • Для внешних пользователей список ограничивается одинаково для прав Чтение и Изменение по одному виду доступа:
    • Внешние пользователи (проверяется значение реквизита Исполнитель);
    • Внешние пользователи (проверяется значение реквизита Автор);
    • достаточно, чтобы хотя бы одно значение было разрешено (в одной из групп доступа).
