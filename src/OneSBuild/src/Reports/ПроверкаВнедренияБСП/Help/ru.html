Отчет ПроверкаВнедренияБСП проверяет корректность внедрения в конфигурацию Библиотеки стандартных подсистем (БСП).

Если установлен параметр отчета Исправлять ошибки, то часть ошибок внедрения исправляется автоматически:
1. Исправление состава разделителей ОбластьДанныхОсновныеДанные и ОбластьДанныхВспомогательныеДанные (только для объектов БСП).
2. Настройка состава ролей ПолныйПрава и АдминистраторСистемы.
3. Установка в ролях стандартных шаблонов ограничений доступа (RLS).
Подробное описание проверок см. в разделах справки к указанным подсистемам:

1. Базовая функциональность

1. Проверяется состав ролей ПолныеПрава и АдминистраторСистемы в соответствии с составом разделителей ОбластьДанныхОсновныеДанные и ОбластьДанныхВспомогательныеДанные. Настраиваются права для объектов, которые могут входить в состав разделителей. Также значения разделителей косвенно вычисляется для Журналов документов и Последовательностей. Для объектов, которые не входят в состав разделителей права не изменяются. В роли ПолныеПрава права для неразделенных объектов не изменяются.

2. Бизнес процессы и задачи

1. Сравнение состава
- БизнесПроцессы.Задание.ВводитсяНаОсновании (за исключением Задача.ЗадачаИсполнителя);
- ОпределяемыеТипы.ПредметЗадачи.Тип.

2. Проверка наличия процедур и функций в модулях менеджеров бизнес процессов (ОпределяемыйТип.БизнесПроцесс)
- Функция ФормаВыполненияЗадачи;
- Процедура ПриПеренаправленииЗадачи;
- Процедура ОбработкаВыполненияПоУмолчанию.

3. Проверка наличия предопределенного элемента ОтветственныйЗаКонтрольИсполнения в справочнике РолиИсполнителей.

4. Проверка наличия предопределенного элемента ВсеОбъектыАдресации в ПВХ ОбъектыАдресацииЗадач

5. Проверка соответствия типов предопределенного объекта ВсеОбъектыАдресации и типов других объектов адресации.

6. Если есть хотя бы один объект адресации, то должна быть роль, в которой используются объекты адресации.

3. Варианты отчетов

1. Проверка заполнения описания вариантов отчетов;

2. Проверка использования устаревших свойств в процедурах ОпределитьНастройкиФормы и ПриОпределенииПараметровВыбора;

3. Проверка корректности подключения к формам отчета.

4. Версионирование объектов

1. Сравнение состава
- ОпределяемыеТипы.ВерсионируемыеДанные.Тип (за исключением Справочник.ИдентификаторыОбъектовМетаданных)
- Объединение типов из ОпределяемыеТипы.ВерсионируемыеДанныеОбъект.Тип (за исключением Справочник.ИдентификаторыОбъектовМетаданных) и подписок ЗаписатьВерсиюДокумента.

2. Проверка наличия в основной форме объекта вставки кода ВерсионированиеОбъектов.ПриСозданииНаСервере.

5. Даты запрета изменения

1. Сравнение состава
- ДатыЗапретаИзмененияПереопределяемый.ЗаполнитьИсточникиДанныхДляПроверкиЗапретаИзменения;
- Подписки с именем вида ПроверитьДатуЗапретаИзмененияПередЗаписью

2. Проверка наличия в основной форме объекта вставки кода ДатыЗапретаИзменения.ОбъектПриЧтенииНаСервере.

6. Дополнительные отчеты и обработки

1. Проверка наличия в основной форме объекта и в основной форме списка вставки кода ДополнительныеОтчетыИОбработки.ПриСозданииНаСервере.

2. Проверка наличия команд открытия дополнительных отчетов и обработок в разделах командного интерфейса.

3. Проверка наличия вставки кода ДополнительныеОтчетыИОбработкиКлиент.ОткрытьФормуКомандДополнительныхОтчетовИОбработок в модулях команд.

7. Заметки пользователя

1. Сравнение состава
- ОпределяемыеТипы.ПредметЗаметок.Тип (за исключением Справочник.ИдентификаторыОбъектовМетаданных);
- ОпределяемыеТипы.ПредметЗаметокОбъект.Тип (за исключением Справочник.Пользователи);
- Типы из подписок с обработчиками ЗаметкиПользователя.УстановитьСтатусИзмененияПометкиУдаленияОбъекта и ЗаметкиПользователя.УстановитьСтатусИзмененияПометкиУдаленияДокумента (за исключением Справочник.Пользователи).

8. Заполнение объектов

1. Ищутся объекты, имеющие в модуле менеджера процедуру ДобавитьКомандыЗаполнения. Затем для каждого такого объекта проверяется наличие хотя бы одной формы со вставками кода ПриСозданииНаСервере, Подключаемый_ВыполнитьКоманду и Подключаемый_ВыполнитьКомандуНаСервере. Затем проверяется наличие хотя бы одной команды, подключенной к этому объекту из отчетов и обработок конфигурации и расширений.

9. Запрет редактирования реквизитов объектов

1. Проверка наличия вставок кода в объектах из ЗапретРедактированияРеквизитовОбъектовПереопределяемый.ПриОпределенииОбъектовСЗаблокированнымиРеквизитами
- В модуле менеджера проверяется наличие функции ПолучитьБлокируемыеРеквизитыОбъекта
- В основной форме объекта проверяется наличие вызова ЗапретРедактированияРеквизитовОбъектов.ЗаблокироватьРеквизиты и процедуры Подключаемый_РазрешитьРедактированиеРеквизитовОбъекта

10. Контактная информация

1. Проверка предопределенных видов контактной информации
- Проверка соответствия имен шаблонам;
- Проверка существования справочников и документов, заданных в качестве владельцев.

2. Сравнение состава владельцев контактной информации
- Предопределенные виды контактной информации;
- ОпределяемыеТипы.ВладелецКонтактнойИнформации.Тип и подписки с обработчиком УправлениеКонтактнойИнформацией.ОбработкаЗаполненияКонтактнойИнформацииДокумента.

3. Проверка табличной части КонтактнаяИнформация объекта-владельца
- Проверка наличия табличной части;
- Проверка наличия обязательных реквизитов;
- Проверка типов реквизитов табличной части.

44. Проверка наличия вставок кода в основной форме объекта-владельца
- УправлениеКонтактнойИнформацией.ПриСозданииНаСервере;
- УправлениеКонтактнойИнформацией.ПриЧтенииНаСервере;
- УправлениеКонтактнойИнформацией.ОбработкаПроверкиЗаполненияНаСервере;
- УправлениеКонтактнойИнформацией.ПередЗаписьюНаСервере;
- УправлениеКонтактнойИнформацией.ОбновитьКонтактнуюИнформацию;
- УправлениеКонтактнойИнформацией.ПослеЗаписиНаСервере (для объектов контактной информацией в табличной части);
- УправлениеКонтактнойИнформациейКлиент.ПриИзменении;
- УправлениеКонтактнойИнформациейКлиент.НачалоВыбора;
- УправлениеКонтактнойИнформациейКлиент.Очистка;
- УправлениеКонтактнойИнформациейКлиент.ВыполнитьКоманду;
- УправлениеКонтактнойИнформациейКлиент.АвтоПодборАдрес;
- УправлениеКонтактнойИнформациейКлиент.ОбработкаВыбора.

11. Напоминания пользователя

1. Сравнение состава
- ОпределяемыеТипы.ПредметНапоминания.Тип (за исключением Справочник.ИдентификаторыОбъектовМетаданных);
- ОпределяемыеТипы.ПредметНапоминанияОбъект.Тип (за исключением Справочник.ИдентификаторыОбъектовМетаданных).

12. Настройка порядка элементов

1. Сравнение состава
- ОпределяемыеТипы.ОбъектСНастраиваемымПорядком.Тип (за исключением Справочник.ИдентификаторыОбъектовМетаданных);
- Объекты с реквизитом РеквизитДопУпорядочивания.

2. Проверка наличия в основной форме списка вставки кода ПодключаемыеКоманды.ПриСозданииНаСервере для объектов из ОпределяемыеТипы.ОбъектСНастраиваемымПорядком.Тип (за исключением Справочник.ВариантыОтветовАнкет и Справочник.ИдентификаторыОбъектовМетаданных).
 
13. Обмен данными

1. Проверка наличия правил регистрации и формы настройки узла.

2. Проверка состава общих команд.

3. Проверка корректности заполнения составов планов обмена распределенной информационной базы.

4. Проверка префикса информационной базы по умолчанию.

14. Обновление версии ИБ

1. Проверка наличия общего макета ОписаниеИзмененийСистемы.

2. Контроль заполнения обязательных свойств отложенных обработчиков обновления.

3. Проверка корректности состава плана обмена ОбновлениеИнформационнойБазы.
 
15. Подключаемые команды

Проверка вставки кода с учетом наличия в конфигурации подсистем Варианты отчетов, Дополнительные отчеты и обработки, Заполнение объектов и Печать.

1. Проверка состава объектов, подключенных в подсистеме ПодключаемыеОтчетыИОбработки, и возможности получения их настроек.

2. Проверка наличия процедур в модулях менеджеров объектов:
- ДобавитьКомандыПечати,
- ДобавитьКомандыОтчетов,
- ДобавитьКомандыЗаполнения.

3. Проверка наличия вызовов в модулях форм:
- ПодключаемыеКоманды.ПриСозданииНаСервере,
- ПодключаемыеКомандыКлиент.ВыполнитьКоманду,
- ПодключаемыеКоманды.ВыполнитьКоманду,
- ПодключаемыеКомандыКлиентСервер.ОбновитьКоманды.

16. Пользователи

1. Сравнение состава
- ОпределяемыеТипы.ВнешнийПользователь.Тип;
- ОпределяемыеТипы.ВнешнийПользовательОбъект.Тип (за исключением Справочник.ВнешниеПользователи);
- Справочники.ВнешниеПользователи.Команды.ВнешнийДоступ.ТипПараметраКоманды.

17. Префиксация объектов

1. Проверка наличия подписок установки префикса.

2. Проверка различия типов подписок с обработчиками
- ПрефиксацияОбъектовСобытия.УстановитьПрефиксОрганизации;
- ПрефиксацияОбъектовСобытия.УстановитьПрефиксИнформационнойБазы;
- ПрефиксацияОбъектовСобытия.УстановитьПрефиксИнформационнойБазыИОрганизации.

18. Работа с файлами

1. Проверка соответствия справочников присоединенных файлов владельцам присоединенных файлов.

2. Сравнение состава типов справочников присоединенных файлов
- ОпределяемыеТипы.ПрисоединенныйФайл.Тип;
- ОпределяемыеТипы.ПрисоединенныйФайлОбъект.Тип (за исключением Справочник.ИдентификаторыОбъектовМетаданных);
- Подписки с обработчиком ПрисоединенныеФайлыКлиентСервер.ПереопределитьПолучаемуюФормуПрисоединенногоФайла;
- Справочники с именем, заканчивающимся на "ПрисоединенныеФайлы".

3. Сравнение состава типов владельцев присоединенных файлов
- ОпределяемыеТипы.ВладелецПрисоединенныхФайлов.Тип (за исключением Справочник.ИдентификаторыОбъектовМетаданных)
- ОпределяемыеТипы.ВладелецПрисоединенныхФайловОбъект.Тип (за исключением Справочник.ИдентификаторыОбъектовМетаданных) и подписки с обработчиком ПрисоединенныеФайлы.УстановитьПометкуУдаленияПрисоединенныхФайловДокументов;
- Владельцы присоединенных файлов, вычисленные по именам справочников, содержащим "ПрисоединенныеФайлы".

4. Проверка реквизитного состава справочников присоединенных файлов
- Проверка наличия обязательных реквизитов;
- Проверка и исправление типов реквизитов справочников;
- Проверка и исправление значений свойств реквизитов справочников.

5. Проверка размещения элементов управления присоединенными файлами на формах.

19. Работа в модели сервиса

1. Для объектов БСП проверяется состав общих реквизитов (разделителей) ОбластьДанныхОсновныеДанные и ОбластьДанныхВспомогательныеДанные.

2. Проверка базовых свойств разделителей: Автоиспользование, Разделение данных и т.п.

3. Проверка объектов, для которых значение разделителя совпадает со значением Авто разделителя (избыточное включение в состав разделителя).

20. Рассылка отчетов

1. Проверка соответствия соответствия объектов из состава ОпределяемыеТипы.ПолучательРассылки.Тип и наличия для них групп контактной информации.

21. Свойства

1. Проверка предопределенных наборов свойств
- Проверка соответствия имен шаблонам;
- Проверка существования объектов метаданных, заданных в качестве владельцев.

2. Проверка соответствия состава типов.

3. Проверка табличной части ДополнительныеРеквизиты объекта-владельца:
- Проверка наличия табличной части;
- Проверка наличия обязательных реквизитов;
- Проверка типов реквизитов табличной части.

4. Проверка наличия вставок кода в основной форме объекта-владельца:
- УправлениеСвойствамиКлиент.РедактироватьСоставСвойств;
- УправлениеСвойствами.ПриСозданииНаСервере;
- УправлениеСвойствамиКлиент.ОбрабатыватьОповещения;
- ОбновитьЭлементыДополнительныхРеквизитов;
- УправлениеСвойствами.ОбновитьЭлементыДополнительныхРеквизитов;
- УправлениеСвойствами.ПриЧтенииНаСервере;
- УправлениеСвойствами.ОбработкаПроверкиЗаполнения;
- УправлениеСвойствами.ПередЗаписьюНаСервере.

22. Структура подчиненности

1. Сравнение состава:
- ОбщиеКоманды.СтруктураПодчиненности.ТипПараметраКоманды;
- КритерииОтбора.СвязанныеДокументы.Тип.

1. Проверка корректности заполнения процедуры ПриЗаполненииВидовОграниченийПравОбъектовМетаданных общего модуля УправлениеДоступомПереопределяемый.

2. Проверка правильности заполнения списка объектов в процедуре УправлениеДоступомПереопределяемый.ПриЗаполненииСписковСОграничениемДоступа.

3. Проверка наличия процедуры ПриЗаполненииОграниченияДоступа в модулях менеджеров объектов.

4. Проверка и исправление текстов шаблонов ограничений прав доступа в ролях по сравнению с поставляемыми.

5. Проверка текстов ограничений прав доступа в ролях.

6. Проверка наличия вызова УправлениеДоступом.ПриЧтенииНаСервере в формах объектов.

7. Проверка и исправление состава определяемых типов:
