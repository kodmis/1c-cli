Документ предназначен для примера использования ряда подсистем.

В подсистеме Базовая функциональность документ показывает пример копирования и вставки строк табличной части Товары с помощью процедур
СкопироватьСтрокиВБуферОбмена, СтрокиИзБуфераОбмена общего модуля ОбщегоНазначения. Копирование также можно выполнять между разными документами, содержащими табличную часть Товары:
Демо: Оприходование товаров
Демо: Перемещение товаров
Демо: Поступление товаров
Демо: Реализация товаров

В подсистеме Даты запрета изменения документ показывает пример запрета изменения документа по разделу дат запрета Демо: Складской учет с объектом Демо: Место хранения.

В подсистеме Дополнительные отчеты и обработки документы используются в качестве примера того, как прикреплять к объектам конфигурации команды формирования внешних печатных форм, а также команды по заполнению, созданию связанных объектов и по формированию дополнительных отчетов.

В подсистеме Печать документ показывает пример реализации печатной формы, размещенной в обработке _ДемоПечатнаяФорма (приведен устаревший способ - для иллюстрации обратной совместимости; для штатного способа внедрения см. документ Демо: Счет на оплату покупателю).

В подсистеме Префиксация объектов к номеру документа добавляются префиксы организации и информационной базы.

В подсистеме Управление доступом документ показывает пример реализации ограничения доступа разного для прав Чтение и Изменение. Пример записи наборов значений доступа для журнала документов Демо: Складские документы.

  • Чтение документов списка ограничивается по двум видам доступа:
    • Организации (проверяется значение реквизита Организация);
    • Демо: Места хранения (проверяется значение реквизита Место хранения);
    • все значения должны быть разрешенными одновременно (в одной из групп доступа).
  • Изменение документов списка ограничивается по трем видам доступа:
    • Организации (проверяется значение реквизита Организация);
    • Демо: Места хранения (проверяется значение реквизита Место хранения);
    • Пользователи (проверяется значение реквизита Ответственный);
    • все значения должны быть разрешенными одновременно (в одной из групп доступа).
