Документ предназначен для примера использования ряда подсистем.

В подсистеме Склонение представлений объектов документ показывает использование склонения ФИО по падежам. При изменении реквизита Физическое лицо автоматически заполняется реквизит Выдать значением ФИО физического лица в дательном падеже.

В подсистеме Управление доступом документ показывает пример использования логики ограничения по типу документа.

  • Тип документа определяется значением реквизита Хозяйственная операция. В случаях когда в каком-нибудь профиле используется только один тип документа требуется скрывать в списке документы остальных типов. Для этого необходимо ввести вспомогательный вид доступа. В данном случае это вид доступа Демо: Хозяйственные операции. Такой вид доступа добавляется в профиль, как предустановленный вид доступа с одним или несколькими разрешенными значениями - типами документов, например, Выдача денежных средств подотчет.
  • Список ограничивается по трем видам доступа:
    • Организации (проверяется значение реквизита Касса.Владелец);
    • Демо: Кассы (проверяется значение реквизита Касса);
    • Демо: Хозяйственные операции (проверяется значение реквизита Хозяйственная операция);
    • все значения должны быть разрешенными одновременно (в одной из групп доступа).

 

В подсистеме Печать документ показывает пример управления составом команд печати в зависимости от выбранной хозяйственной операции.
