Документ предназначен для примера использования ряда подсистем.

В подсистеме Банки документ показывает пример работы с информацией о банковском счете.

В подсистеме Версионирование объектов документ демонстрирует историю версий документа.

В подсистеме Дополнительные отчеты и обработки документ используются в качестве примера того, как прикреплять к объектам конфигурации команды формирования внешних печатных форм.

В подсистеме Загрузка данных из файла документ показывает пример загрузки номенклатуры из файлов в табличную часть Товары, используя предварительно сохраненный бланк для загрузки.

В подсистеме Печать документ демонстрирует возможность формирования печатных форм в различных форматах (табличный документ, Office Open XML, PDF). Команда Счет на оплату (комплект) демонстрирует печать комплекта документов, без возможности сохранения числа копий. Команда Счет на оплату демонстрирует использование факсимильной подписи и печати в печатных формах.

В подсистеме Работа с файлами документ показывает пример сохранения печатной формы в качестве присоединенного файла. Для хранения файлов-вложений используется справочник Присоединенные файлы (Демо: Счет на оплату покупателю). Кроме того, в форме элемента приведен пример отображения гиперссылки присоединенных файлов.

В подсистеме Структура подчиненности документ показывает пример иерархической взаимосвязи документов (на этот документ ссылается реквизит Счет табличной части СчетаНаОплату документа Демо: Заказ покупателя).

В подсистеме Текущие дела документ выступает в качестве поставщика текущих дел по счетам на оплату для демонстрации работы в сеансе внешнего пользователя.

В подсистеме Управление доступом документ показывает пример различий в ограничениях для сеансов разного назначения (для пользователей и для внешних пользователей).

  • Для пользователей список ограничивается по двум видам доступа:
    • Организации (проверяется значение реквизита Организация).
    • Демо: Группы партнеров (проверяется значение реквизита Партнер).
  • Для внешних пользователей список ограничивается по других двух видам доступа:
    • Внешние пользователи (проверяется значение реквизита Партнер);
    • Внешние пользователи (проверяется значение реквизита Ссылка элемента справочника Внешние пользователи, у которого значение реквизита Объект авторизации = значению реквизита Ссылка элемента справочника Демо: Контактные лица, у которого в свою очередь значение реквизита Владелец = значению реквизита Партнер элемента этого списка);
    • достаточно, чтобы значение было разрешено в рамках любой из двух проверок по виду доступа (в одной из групп доступа).
