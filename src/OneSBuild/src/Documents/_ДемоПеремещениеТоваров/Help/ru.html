В подсистеме Даты запрета изменения документ показывает пример запрета изменения документа по разделу дат запрета Демо: Складской учет с двумя объектами Демо: Место хранения, которые получаются из двух реквизитов Место хранения источник и Место хранения приемник.

В подсистеме Префиксация объектов к номеру документа добавляются префиксы организации и информационной базы.

В подсистеме Управление доступом документ показывает типичный пример логики ограничения для документов перемещения.
В данном случае логика ограничения для права Чтения - это разрешенность любого из мест хранения, а для права Изменения - это разрешенность одновременно двух мест хранения.
Документ записывает наборы значений доступа для журнала документов Демо: Складские документы.

В подсистеме Управление доступом документ показывает пример реализации ограничения доступа разного для прав Чтение и Изменение. Пример записи наборов значений доступа для журнала документов Демо: Складские документы.

  • Чтение документов списка ограничивается по двум видам доступа:
    • Организации (проверяется значение реквизита Организация);
    • Демо: Места хранения (проверяется значение реквизита Место хранения источник);
    • Демо: Места хранения (проверяется значение реквизита Место хранения приемник);
    • Должна быть разрешена организация и одно из мест хранения (в одной из групп доступа).
  • Изменение документов списка ограничивается по трем видам доступа:
    • Организации (проверяется значение реквизита Организация);
    • Демо: Места хранения (проверяется значение реквизита Место хранения источник);
    • Демо: Места хранения (проверяется значение реквизита Место хранения приемник);
    • Пользователи (проверяется значение реквизита Ответственный);
    • все значения должны быть разрешенными одновременно (в одной из групп доступа).
