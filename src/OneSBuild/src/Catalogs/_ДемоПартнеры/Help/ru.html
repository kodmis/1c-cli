Справочник предназначен для демонстрации работы ряда подсистем.

В подсистеме Анкетирование элементы справочника выступают в качестве респондентов, имеющих доступ к информационной базе как внешние пользователи.

В подсистеме Взаимодействия справочник демонстрирует пример контакта взаимодействий.

В подсистеме Даты запрета изменения для элементов справочника может быть установлен запрет на редактирование в разрезе раздела Торговля с объектами Демо: Группы доступа партнеров.

В подсистеме Контактная информация элементы справочника являются объектами-владельцами контактной информации.

В подсистеме Пользователи элементы справочника выступают в качестве объектов авторизации внешних пользователей, имеющих доступ к информационной базе для работы с задачами, анкетами, счетами на оплату и другими объектами и подсистемами, которые предназначены для внешних пользователей.

В подсистеме Префиксация объектов к коду элементов справочника добавляется префикс информационной базы.

В подсистеме Работа с файлами элементы справочника используются в качестве объектов-владельцев файлов вложений. Для хранения файлов-вложений используется общий справочник Файлы. Демонстрируется возможность вывода уведомления о захваченных файлах в момент закрытия формы, если в списке файлов остались занятые для редактирования. Кроме того, в форме элемента приведен пример отображения гиперссылки присоединенных файлов.

В подсистеме Свойства элементы справочника являются объектами-владельцами дополнительных реквизитов и сведений. При этом набор дополнительных свойств для конкретного элемента справочника зависит от значений реквизитов Клиент, Поставщик, Конкурент и Прочие отношения. В списке показан пример вывода дополнительных реквизитов для произвольных объектов.

В подсистеме Управление доступом элементы справочника являются значениями доступа.

  • Значения доступа Демо: Партнеры назначаются в группах доступа в списке разрешенных значений по виду доступа Демо: Группы партнеров с помощью элементов Демо: Группы доступа партнеров.
  • Ограничение по виду доступа Демо: Группы партнеров используется в списках Демо: Партнеры, Демо: Контактные лица партнеров, Демо: Контрагенты, Демо: Договоры контрагентов, Демо: Банковские счета, Демо: Группы доступа партнеров, Демо: Заказ покупателя, Демо: Поступление товаров, Демо: Реализация товаров, Демо: Счет-фактура полученный, Демо: Счет на оплату покупателю, Присоединенные файлы (Демо: Счет на оплату покупателю).
  • Для пользователей список ограничивается по одному виду доступа:
    • Демо: Группы партнеров (проверяется значение реквизита Ссылка).
  • Для внешних пользователей список ограничивается по одному виду доступа:
    • Внешние пользователи (проверяется значение реквизита Ссылка);
    • Внешние пользователи (проверяется значение реквизита Ссылка элемента справочника Внешние пользователи, у которого значение реквизита Объект авторизации = значению реквизита Ссылка элемента справочника Демо: Контактные лица, у которого в свою очередь значение реквизита Владелец = значению реквизита Ссылка элемента этого списка);
    • достаточно, чтобы значение было разрешено в рамках любой из двух проверок по виду доступа (в одной из групп доступа).
  • В списке ограничиваются только элементы, группы элементов (папки) не ограничиваются.

 В подсистеме Поиск и удаление дублей форма списка справочника используется для демонстрации внедрения подсистемы.

 В подсистеме Удаление помеченных объектов форма списка справочника демонстрирует пример встраивание механизма скрытия помеченных на удаление объектов без использования подсистемы Подключаемые команды.
