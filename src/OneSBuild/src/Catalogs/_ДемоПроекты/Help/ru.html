Справочник предназначен для демонстрации примера использования ряда подсистем.

В подсистеме Базовая функциональность справочник демонстрирует установку параметра сеанса и формирование заголовка главного окна программы. См. команду Сделать основным контекстного меню списка проектов.

В подсистеме Бизнес-процессы и задачи элементы справочника используются в качестве объектов адресации задач (при ролевой адресации).

В подсистеме Настройка порядка элементов элементы справочника могут быть упорядочены пользователем.

В подсистеме Префиксация объектов демонстрируется переназначение номера справочника при смене организации.

В подсистеме Работа с файлами элементы справочника используются в качестве объектов-владельцев файлов вложений. Реквизит РегламентПроекта демонстрирует форму выбора присоединенных файлов. В форме списка добавлена колонка со скрепкой - признаком наличия вложений. Для комфортабельного хранения большого количества файлов предусмотрена возможность создавать папки. Кроме того, в форме элемента приведены примеры отображения гиперссылки присоединенных файлов.

В подсистеме Шаблонов сообщений демонстрируется отправка письма по шаблону сообщений, где список вариантов шаблонов сообщений зависит от владельца шаблона Демо: Подразделения.

В подсистеме Управление доступом показывает использование логики ограничения разной для прав Чтение и Изменение.

  • Чтение списка ограничивается по одному виду доступа:
    • Организации (проверяется значение реквизита Организация).
  • Изменение списка ограничивается по одному двум видам доступа:
    • Организации (проверяется значение реквизита Организация).
    • Пользователи (проверяется значение реквизита Ответственный, если заполнен).
