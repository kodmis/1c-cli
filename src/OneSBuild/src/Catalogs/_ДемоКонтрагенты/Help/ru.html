Справочник предназначен для демонстрации использования ряда подсистем.

В подсистеме Анкетирование элементы справочника используются в качестве респондентов, имеющих доступ к базе как внешние пользователи.

В подсистеме Версионирование объектов при элементах справочника сохраняются их версии.

В подсистеме Взаимодействия справочник демонстрирует пример контакта взаимодействий, для которых возможен быстрый поиск (подбор) по ИНН (см. свойство справочника Ввод по строке).

В подсистеме Групповое изменение объектов элементы справочника демонстрируют возможность группового изменения прямо из формы списка, а также в модуле менеджера дополнительно задается список реквизитов-исключений, которые не рекомендуется редактировать с помощью группового изменения реквизитов.

В подсистеме Даты запрета изменения элементы справочника могут быть запрещены от редактирования с помощью установки даты в разрезе раздела Торговля и объектов Демо: Группы доступа партнеров.

В подсистеме Дополнительные отчеты и обработки элементы справочника используются в качестве примера того, как прикреплять к объектам конфигурации команды вызова дополнительных обработок по заполнению и созданию связанных объектов, а также по формированию дополнительных отчетов.

В подсистеме Заполнение объектов элементы справочника используются в качестве примера того, как подключать к объектам конфигурации команды заполнения.

В подсистеме Заметки пользователя элементы справочника являются предметами заметок.

В подсистеме Контактная информация элементы справочника являются объектами-владельцами контактной информации (КИ), по умолчанию предопределены следующие виды контактной информации: адрес, электронная почта, skype.

В подсистеме Напоминания пользователя элементы справочника являются предметами напоминаний.

В подсистеме Обмен данными элементы справочника используются в качестве объектов, синхронизируемых с другими информационными базами.

В подсистеме Отправка SMS элементы справочника демонстрируют возможность отправки SMS контрагенту при наличии у него контактной информации.

В подсистеме Префиксация объектов к коду элементов справочника добавляется префикс информационной базы.

В подсистеме Свойства элементы справочника являются объектами-владельцами дополнительных реквизитов и сведений. При этом демонстрируется отложенная инициализация дополнительных реквизитов, которые загружаются при переключении на закладку "Доп. реквизиты". Данный вариант внедрения позволяется ускорить время открытия формы объекта.
См. также пример внедрения, где для объекта используется больше одного набора свойств - Демо: Партнеры.

В подсистеме Управление доступом роли для доступа к элементам справочника содержат пример ограничения по значению доступа - ссылке на элемент справочника Демо: Партнеры. Справочник также показывает пример различий в ограничениях для пользователей и для внешних пользователей.

  • Для пользователей список ограничивается по одному виду доступа:
    • Демо: Группы партнеров (проверяется значение реквизита Партнер).
  • Для внешних пользователей список ограничивается по одному виду доступа:
    • Внешние пользователи (проверяется значение реквизита Партнер);
    • Внешние пользователи (проверяется значение реквизита Ссылка элемента справочника Внешние пользователи, у которого значение реквизита Объект авторизации = значению реквизита Ссылка элемента справочника Демо: Контактные лица, у которого в свою очередь значение реквизита Владелец = значению реквизита Партнер элемента этого списка);
    • достаточно, чтобы значение было разрешено в рамках любой из двух проверок по виду доступа (в одной из групп доступа).
